# Security Policy

## Supported Versions

Aktif olarak `main` branch desteklenir.
Patch ve guvenlik duzeltmeleri once `main` uzerinden yayinlanir.

## Reporting a Vulnerability

Guvenlik aciklarini public issue olarak paylasmayin.

Iletisim:

- GitHub Security Advisories (tercih edilen)
- veya repository owner ile direkt iletisim

Raporlarken sunlari ekleyin:

- Etkilenen endpoint veya modul
- Tekrarlanabilir adimlar
- Beklenen ve gerceklesen davranis
- Olasi etki seviyesi (dusuk/orta/yuksek/kritik)
- Mumkunse PoC

## Response Targets

- Ilk geri donus: 72 saat
- Degerlendirme + aksiyon plani: 7 gun
- Duzeltme suresi: etki seviyesine gore planlanir
